# World Model Experiments

This folder contains training/evaluation code and run scripts for world-model experiments on maze trajectories.

## Dataset

- Source file: `data.txt` (symlinked in this folder to `../../../Algernon/data.txt`)

### File format

- Episodes are separated by lines like:
  - `random dim=<D> extra_openings=<K> seed=<S>`
- Each following line is a semicolon-separated list of step dicts:
  - `location`: `[x, y]` grid coordinate
  - `heading`: `u/d/l/r` (legacy `up` also appears and is normalized to `u` in training code)
  - `sensor`: `[left, front, right]` integer distances
  - `action`: `(turn_deg, steps)`

### How mazes are generated (from `../../Algernon/genmaze.py` and `tester.py`)

- Maze size `dim` is sampled from even values `12..24`.
- A perfect maze is first created with randomized DFS backtracking.
- Then `extra_openings` are added to create loops.
- In random generation mode:
  - `extra_openings` is sampled in approximately `[dim^1.5, dim^1.8]` (integer range).
  - A per-maze random `seed` is stored in the episode header.

### How trajectories are generated (from `../../Algernon/tester.py` and `robot.py`)

- Trajectories are generated by the built-in maze-solving agent in `robot.py`, which uses flood-fill map updates and A*.
- Robot starts at `(0, 0)` facing `up`.
- Goal region is the center 2x2 block.
- Sensor values are true ray distances to nearest wall (`dist_to_wall`) in relative directions `[left, front, right]` with respect to current heading.
- Logged step is pre-transition data:
  - state at time `t` (`location`, `heading`, `sensor`)
  - chosen `action_t`
  - then environment applies rotation/movement.
- Action semantics:
  - `turn_deg` in `{ -90, 0, 90 }`
  - `steps` clipped to `[-3, 3]`; movement can terminate early if blocked by wall.
- So dataset supervision naturally corresponds to next-step prediction:
  - `(obs_t, action_t) -> obs_{t+1}`.

### Quick dataset stats (current `data.txt`)

- Episodes: `500`
- Total logged steps: `72,471`
- Episode length: min `12`, max `715`, mean `144.94`, median `121`
- Maze dimensions present: `12, 14, 16, 18, 20, 22, 24`
- `extra_openings`: min `41`, max `306`, mean `131.22`
- Action support:
  - `turn_deg`: `{-90, 0, 90}`
  - `steps`: `{0, 1, 2, 3}`
- Sensor value range in file: `[0, 18]`

In current training code:
- Inputs are sensor + normalized action history.
- Targets are next-step sensors.
- Location/heading/turn/step are mainly used as probes/diagnostics (depending on loss weights and mode).

## Model Variants

- `transformer`: deterministic baseline (`UnifiedPredictor`)
- `rnn`: deterministic baseline (`RNNPredictor`, GRU/residual options)
- `rssm-discrete`: latent world model with discrete stochastic state
- `tssm`: transformer state-space model with discrete stochastic state

## Training / Evaluation

- Main trainer: `wm_train.py`
- Reverse-path evaluation: `wm_reverse_path_test.py`
- Speed benchmark: `wm_bench_rssm_tssm_speed.py`
- Canonical full history split: train/val split by episode (not by timestep).

## Recent Experiment Overview (Short)

- Baseline sweeps found `h176/l3` RNN to be near transformer parameter count and a useful default for fair comparison.
- TSSM is much slower than RSSM in our benchmark (roughly ~8x in tested settings), which strongly affects long-run throughput.
- Deterministic baselines can overfit next-sensor prediction; generalization/open-loop reverse-path remains harder.
- Transformer pretraining completed:
  - checkpoint: `outputs/wm_transformer_pretrain_h176l3.pt`
- RNN pretraining tracks (RMSNorm vs none) are active; see latest snapshots in `worklog.md`.
